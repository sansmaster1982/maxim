# Этап 3 — Нативный iOS-слой поведения

Дата: 2026-06-04. Цель: добавить нативное для iOS поведение поверх уже существующей бренд-точной темы, без слепого переписывания вёрстки.

## Проверка темы

Палитра `lib/ui/theme/app_theme.dart` сверена с реверс-цветами десктопного клиента (`maxclient/ui/theme.py`): тёмная `#0F1117`/`#1A1F2B`, светлая `#FFFFFF`/`#EEF0F3`, accent `#0066FF`. Совпадает — прежняя работа уже взяла официальные цвета. Палитру не трогал.

## Сделано

- **Cupertino-переходы по всему приложению**: `pageTransitionsTheme` с `CupertinoPageTransitionsBuilder` для всех `TargetPlatform` в обеих темах (light/dark). Это включает iOS-слайд и жест свайпа-назад от края экрана сразу для ВСЕХ `MaterialPageRoute` — один источник правды вместо правки восьми мест навигации.
- **Адаптивные контролы**:
  - `Switch` → `Switch.adaptive` (профиль) — на iOS рисуется `CupertinoSwitch`.
  - splash-спиннер → `CircularProgressIndicator.adaptive` — на iOS нативный кружок Cupertino.

Скролл-физика менять не нужно: `MaterialApp` на iOS по умолчанию даёт bouncing-физику.

## Честные границы

Пиксель-в-пиксель повторение iOS-облика оригинального MAX (точные отступы, иконки, тонкая типографика SF Pro) требует референс-скриншотов реального приложения и Mac для визуальной итерации — это вне текущего окружения (Windows, без iOS-рендера). Сделан слой поведения (переходы, адаптивные контролы) на бренд-точной палитре; дальнейшая доводка вёрстки — когда появится визуальная обратная связь.
